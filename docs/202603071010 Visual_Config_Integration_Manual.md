## 3. 功能复盘 / Functionality Review

### 3.1 核心功能说明 / Core Functionality
1. **全局控制面板 (Global Dashboard):** 
   - 彻底替换了原先“单篇新闻快照即首页”的僵硬逻辑。
   - 采用 `iframe` 容器技术，在根路径下无缝嵌套展示最新的新闻聚合结果。
   - Completely replaced the rigid logic where a single news snapshot served as the homepage. Uses iframe container to seamlessly display the latest news aggregates at the root path.

2. **浮动配置窗口 (Floating Config Window):**
   - **交互触发 (Trigger):** 报告页右下角常驻蓝绿色圆形悬浮按钮（标题 `可视化配置 / Config`）。
   - **窗口特性 (Window Features):** 
     - **可拖拽 (Draggable):** 通过原生 JS 监听标题栏实现自由移动。
     - **可缩放 (Resizable):** 利用原生 CSS `resize: both` 允许用户自由调节窗口大小。
     - **内嵌集成:** 弹窗内通过 iframe 加载 `config_editor/index.html`，支持 `config.yaml`、`frequency_words.txt`、`timeline.yaml`、AI 分析/翻译提示词五类配置。
   - Interaction: Persistent floating config button at bottom-right. Features: Draggable via header, Resizable via CSS, and iframe-embedded config center.

3. **方案管理功能 (Profile Management):**
   - **保存方案 (Save Profile):** 支持显示已保存的方案列表，用户可选择序号覆盖现有方案或输入新名称保存。覆盖时弹出确认对话框。留空则自动生成时间戳名称。
   - **提取方案 (Load Profile):** 从服务器加载已保存的方案列表，支持一键切换爬取策略。
   - **持久化存储 (Persistence):** 方案以 JSON 对象形式打包所有配置文件，存储在服务器 `profiles/` 目录，自动维持最近 5 个方案。
   - Save Profile: Display existing profiles list, allow selecting by index to overwrite or input new name. Confirmation dialog on overwrite. Auto-generate timestamp if empty.
   - Load Profile: Load saved profiles from server, one-click strategy switching.
   - Persistence: Bundle all configs as JSON, store in server `profiles/` directory, auto-maintain last 5 profiles.

4. **锁定编辑功能 (Lock Edit Mode):**
   - **默认锁定 (Default Locked):** 系统启动时默认为锁定状态，保护配置不被误修改。
   - **细粒度控制 (Fine-grained Control):** 锁定状态下，只禁用修改类按钮（APPLY、SAVE & RUN），保留浏览/加载类按钮可用。
   - **滚动保留 (Scroll Preserved):** 右侧配置面板和左侧编辑器在锁定状态下仍可滚动浏览内容。
   - **全面禁用 (Comprehensive Disable):** 右侧配置面板的所有输入元素（input、select、textarea、button）在锁定状态下显示为禁用（变灰）。
   - Default Locked: System starts in locked state to prevent accidental config changes.
   - Fine-grained Control: Only modify buttons (APPLY, SAVE & RUN) disabled; browse/load buttons remain available.
   - Scroll Preserved: Right panels and left editors remain scrollable in locked state.
   - Comprehensive Disable: All input elements in right panels show disabled state (grayed out) when locked.

5. **频率词可视化编辑 (Frequency Words Visual Editor):**
   - 支持 `[GLOBAL_FILTER]` 与 `[WORD_GROUPS]` 两个区域解析。
   - 词组类型包括：组别名、多关键词普通组、单个别名、连续别名组。
   - 连续别名组以无空行的 `关键词/正则 => 显示别名` 行组成，同一组内支持分类标签 `@tags:热榜,科技`。
   - 连续别名组卡片的“别名列表”下方必须提供 `+ 添加别名` 按钮，点击后在同一卡片中新增空白别名项；空白项只保留在 UI 中等待填写，不应写入 `frequency_words.txt`。

6. **AI 模型查询与预设 (AI Model Query & Presets):**
   - AI 配置模块支持模型查询、连接测试、可用模型列表、搜索过滤、自动 provider 前缀补全。
   - 后端支持 `/api/check_ai_connection` 与 `/api/get_ai_models`，并兼容 `/models` 与 `/v1/models`。
   - 自定义模型预设保存在浏览器 `localStorage` 中，不写入服务器方案文件，除非用户把当前配置保存为方案。

#### 复盘范围 / Scope

#### 问题 0: 迁移后默认配置与 AI 窗口缺失 / Defaults and AI Card Missing After Migration
- **问题现象:** 新服务器部署后，配置中心可能只显示 `config.yaml`、`frequency_words.txt`、`timeline.yaml` 三个文件；`加载默认` 缺少 AI 提示词默认文件；未配置大模型或 API Key 时，HTML 报告中的 `✨ AI 深度分析研判` 卡片消失。
- **根因:** 迁移备份只保留运行配置中的部分文件，漏带 `docs/defaults/ai_analysis_prompt.txt` 和 `docs/defaults/ai_translation_prompt.txt`；旧启动脚本只在 `output/config_editor/index.html` 不存在时复制配置中心，导致旧 3-tab 运行副本不会被刷新；HTML V2 渲染只在 `ai_analysis` 对象存在时生成 AI 卡片。
- **实现路径:** 在前端 `CORE_CONFIG_FILES` 固定五个核心文件，`加载默认`、版本检测和默认更新统一走该清单；`docs/defaults/` 与 `output/config_editor/defaults/` 固定保存五个只读默认模板；Docker 镜像复制 `docs/defaults/`，`entrypoint.sh` 每次启动同步配置中心静态文件；`html_v2.py` 在 `ai_analysis is None` 或失败时渲染配置引导卡片。
- **强制约束:** `docs/defaults/` 是内置只读模板；部署者运行配置只写入 `config/`；用户加载默认后只能在编辑器中修改并保存成自己的配置，不得写回默认模板；五个核心文件不得从分页签、默认加载清单、Docker defaults 或迁移校验中移除。
- **验证方式:** 运行 `node --check docs/assets/script.js`、`node --check output/config_editor/assets/script.js`、`python3 -m py_compile aiyxdata_tradar/report/html_v2.py`、`bash -n scripts/export_clean_repo.sh`；用 `render_html_content_v2(..., ai_analysis=None)` 断言输出包含 `✨ AI 深度分析研判`、`可视化配置中心` 和 `API Key`。

#### 问题 1: 连续别名组只能删除不能新增 / Alias Group Add Missing
- **问题现象:** `frequency_words.txt` 可视化编辑中，连续别名组卡片只有每行删除按钮，没有位于别名列表下方的 `+ 添加别名` 按钮，用户无法在同一连续别名组内继续添加别名行。
- **根因:** 连续别名组渲染逻辑只覆盖已有 `items` 的编辑/删除，没有提供新增入口；同时新增空白项如果直接落盘会生成非法 ` => ` 行。
- **实现路径:** 生产配置中心逻辑位于 `/TrendRadar/output/config_editor/assets/script.js`，核心函数包括 `renderGroupCard()`、`addAliasItem(groupIndex)`、`syncAliasGroupEdit()`、`buildFrequencyText(data)`；静态参考副本如需维护，同步 `/TrendRadar/docs/assets/script.js`。
- **最终表现:** 连续别名组卡片显示标签 `连续别名组`，别名列表下方、`AI 写正则` 行之前固定出现 `+ 添加别名`；点击后在同一张卡片新增空白别名项。
- **强制约束:** 空白新增项只存在于 UI 状态中，`buildFrequencyText()` 不得把空 `keyword` 输出到 `frequency_words.txt`；删除到 0 项删除整组，删除到 1 项可降级为单个别名。
- **验证方式:** 运行 `node --check output/config_editor/assets/script.js`；打开报告页配置中心，进入 `frequency_words.txt`，确认连续别名组卡片可新增、删除、编辑，并且 APPLY 后不产生空 ` => ` 行。

#### 问题 2: 左侧五个文件标签显示不全且排版生硬 / Five File Tabs Layout
- **问题现象:** 配置中心左侧文件标签栏中，`ai_translation_prompt.txt` 等长文件名会被遮挡或挤出，用户无法同屏看清五个文件入口；早期压缩后文字仍被省略，视觉上不像稳定分页签。
- **根因:** 文件标签与保存状态共用同一行宽度，长英文文件名按单行展示，缺少两行结构和自适应字号。
- **实现路径:** 运行入口是 `/TrendRadar/output/config_editor/index.html`；样式在 `/TrendRadar/output/config_editor/assets/style.css`；交互与自适应在 `/TrendRadar/output/config_editor/assets/script.js` 的 `fitFileTabs()` 及文件切换逻辑。
- **最终表现:** 五个文件标签采用分页签形式并同屏可见；上方显示英文文件名，下方显示中文说明；保存状态移动到独立 `.file-savebar`，不再挤占标签行。
- **强制约束:** 不得为解决标签栏问题改动读取、保存、APPLY、SAVE & RUN 或右侧可视化面板逻辑；英文文件名不得被省略号截断；五个文件入口必须同时可见。
- **验证方式:** 运行 `node --check output/config_editor/assets/script.js`；刷新 `http://127.0.0.1:8084/config_editor/index.html` 或报告弹窗 iframe，确认 `config.yaml`、`frequency_words.txt`、`timeline.yaml`、`ai_analysis_prompt.txt`、`ai_translation_prompt.txt` 全部可见且上下两行显示。

#### 问题 3: 关键词组分类标签必须强制且同步到报告 / Mandatory Keyword Group Tags
- **问题现象:** 部分关键词组没有分类标签，导致报告卡片标题区缺少分类信息；仅靠前端提示不能防止直接 API 或保存方案绕过。
- **根因:** `frequency_words.txt` 中 `@tags:` 原本只是可选元数据，解析、分析、报告生成链路没有完整传递；后端 `/api/save` 没有强制校验。
- **实现路径:** 前端强制逻辑在 `/TrendRadar/output/config_editor/assets/script.js`，包括 `FREQUENCY_GROUP_TAGS`、`validateFrequencyGroupTags(content)`、`getGroupsMissingRequiredTags()`、`focusFrequencyGroup()`；后端兜底在 `/TrendRadar/docker/server.py` 的 `validate_frequency_required_tags(content)`；解析和报告链路在 `/TrendRadar/aiyxdata_tradar/core/frequency.py`、`/TrendRadar/aiyxdata_tradar/core/analyzer.py`、`/TrendRadar/aiyxdata_tradar/report/generator.py` 与 `/TrendRadar/aiyxdata_tradar/report/html_v2.py`。
- **最终表现:** 每个关键词组卡片顶部动态显示已选标签；无标签时显示 `必须选择标签` 状态；多个标签全部显示，不区分选择顺序；报告中的关键词卡片同步显示该组所有标签。
- **强制约束:** 每个 `[WORD_GROUPS]` 词组至少一个标签；APPLY、SAVE & RUN、保存方案和 `/api/save file=frequency` 都必须拒绝无标签内容；后端校验不能被前端状态绕过。
- **验证方式:** 运行 `node --check output/config_editor/assets/script.js`、`python3 -m py_compile docker/server.py aiyxdata_tradar/core/frequency.py aiyxdata_tradar/core/analyzer.py aiyxdata_tradar/report/generator.py aiyxdata_tradar/report/html_v2.py`；用 `/api/save file=frequency` 测试无标签内容应失败、有标签内容应成功。

#### 问题 4: 热榜平台和 RSS 独立卡片缺少固定来源标签 / Fixed Source Tags
- **问题现象:** 报告页中来自 `# 2. 数据源 - 热榜平台` 和 `# 3. 数据源 - RSS 订阅` 的独立卡片没有在标题区显示来源类别标签。
- **根因:** `/TrendRadar/aiyxdata_tradar/report/html_v2.py` 只对 `keyword` 类型卡片渲染 `tags`，`source` 和 `rss` 类型卡片未携带标签。
- **实现路径:** 在 `html_v2.py` 构建 `all_blocks` 时，`source` 类型写入 `tags: ["热榜"]`，`rss` 类型写入 `tags: ["RSS源"]`；渲染逻辑统一读取 `block.get("tags", [])`，不再只限制关键词卡片。
- **最终表现:** 热榜平台独立卡片固定显示 `热榜`，RSS 独立卡片固定显示 `RSS源`；该标签按来源类型自动生成，不随单个源名称手动维护。
- **强制约束:** 不得把 `热榜`/`RSS源` 写成具体源名白名单；无论未来新增、删除或重命名 `platforms.sources` / `rss.feeds` 中的源，只要进入对应卡片类型，就必须自动显示对应固定标签。
- **验证方式:** 运行 `python3 -m py_compile aiyxdata_tradar/report/html_v2.py`；用 `render_html_content_v2()` 构造最小 `standalone_data`，断言 HTML 中包含 `<span class="card-category-tag">热榜</span>` 与 `<span class="card-category-tag">RSS源</span>`；当前静态输出可用 `rg` 或 `curl http://127.0.0.1:8084/` 检查。

#### 问题 5: 数据源还需要手工分类标签且必须强制 / Manual Source Tags
- **问题现象:** 仅有固定来源标签时，报告能区分 `热榜` / `RSS源`，但无法表达每个源的业务分类；如果不强制，新增热榜平台或 RSS 源仍可能出现没有分类标签的卡片。
- **根因:** `config.yaml` 的 `platforms.sources[]` 与 `rss.feeds[]` 之前没有统一的 `tags` 字段，也缺少前后端保存校验；固定来源标签被误当成完整分类信息。
- **实现路径:** 在配置中心第 2/第 3 节列表项中加入固定标签 chip、手工标签 chip 和 `标签` 多选下拉；手工标签写入每个源的 `tags: [...]`；前端 `validateSourceRequiredTags()` 在本地保存、APPLY、SAVE & RUN、保存方案前校验；后端 `/TrendRadar/docker/server.py` 的 `validate_source_required_tags()` 在 `/api/save file=config` 与 `/api/profiles/save` 再校验；`__main__._prepare_standalone_data()` 将源配置 tags 带入 `standalone_data`，`html_v2.py` 合并固定标签和手工标签。
- **最终表现:** 热榜平台行固定显示 `热榜`，RSS 行固定显示 `RSS源`；每行必须再选择至少一个手工标签，可多选，选择后列表行实时显示；报告卡片标题区同步显示固定标签 + 所有手工标签。
- **强制约束:** 固定标签不计入必选。每个 `platforms.sources[]` / `rss.feeds[]` 都必须有至少一个合法手工标签，否则 APPLY、SAVE & RUN、保存方案、`/api/save file=config` 都必须失败并定位缺失行；新增源允许先创建为空标签，但在保存到后端前必须补齐。
- **配置落点:** 现有 `config/config.yaml` 中 11 个 `platforms.sources[]` 和 8 个 `rss.feeds[]` 均已补 `tags: [...]`；对比 HEAD 去掉 `tags` 后语义必须保持一致，避免借补标签误改数据源。
- **验证方式:** 运行 `node --check output/config_editor/assets/script.js`、`python3 -m py_compile docker/server.py aiyxdata_tradar/__main__.py aiyxdata_tradar/report/html_v2.py`；用当前 `config/config.yaml` 调用 `validate_source_required_tags()` 应成功，删除任一源 `tags` 后应失败；HTTP 正反用例必须覆盖 `/api/save file=config`：缺标签返回 `400` 且错误含 `缺少手工分类标签`，合法配置返回 `200`；`/api/profiles/save` 的缺标签 bundle 也必须返回 `400`；用最小 `standalone_data` 渲染 HTML，断言同一卡片同时包含 `热榜`/`RSS源` 和手工标签。

#### 问题 6: 运行路径与静态参考路径混淆 / Runtime Path Confusion
- **问题现象:** 修改配置中心相关问题时，容易只改 `/TrendRadar/docs/assets/script.js` 或历史静态页面，但报告弹窗中的配置中心仍不变化。
- **根因:** 报告页配置弹窗通过 iframe 加载 `output/config_editor/index.html`，生产运行脚本是 `output/config_editor/assets/script.js`，`docs/` 只是静态参考/GitHub Pages 版本。
- **实现路径:** 本文档已将 `/TrendRadar/output/config_editor/` 明确为生产配置中心运行副本；`/TrendRadar/docs/` 只作为参考副本。后端 API 与静态服务来自 `/TrendRadar/docker/server.py`，容器内通过挂载映射到 `/app/server.py`、`/app/output`、`/app/config`、`/app/aiyxdata_tradar`。

#### 问题 7: Docker 双入口导致后端校验可能跑旧代码 / Docker Server Entry Drift
- **问题现象:** 本地直接加载 `/TrendRadar/docker/server.py` 时，`validate_source_required_tags()` 能正确拒绝缺标签配置；但 HTTP `/api/save file=config` 曾经仍返回 `200`，说明真实容器进程没有执行新版后端校验。
- **根因:** 当前容器镜像可能保留旧入口 `WORKDIR=/app/docker` 且 `CMD ["python3", "server.py"]`，而 compose 早期只把 `/TrendRadar/docker/server.py` 挂载到 `/app/server.py`。这会让 `python3 server.py` 从 `/app/docker/server.py` 启动旧镜像内文件，形成 `/app/server.py` 与 `/app/docker/server.py` 双版本。
- **实现路径:** `/TrendRadar/docker/docker-compose.yml` 强制 `working_dir: /app`，并把 `./server.py` 同时挂载到 `/app/server.py` 与 `/app/docker/server.py`；`/TrendRadar/docker/Dockerfile` 构建阶段也复制 `docker/server.py` 到 `/app/server.py`，保证新构建镜像有同一入口；重建服务使用 `docker-compose -f docker/docker-compose.yml up -d aiyxdata_tradar`。
- **最终表现:** `docker inspect aiyxdata_tradar` 显示 `WorkingDir=/app`，挂载列表同时包含 `/TrendRadar/docker/server.py -> /app/server.py` 与 `/TrendRadar/docker/server.py -> /app/docker/server.py`；容器内两个路径 inode 一致或内容一致；HTTP `/api/save` 实际执行新版 `validate_source_required_tags()`。

### 3.2.1 Nginx 反向代理配置 / Nginx Reverse Proxy Configuration

#### 配置文件位置 / Configuration File Location
- **路径:** `/etc/nginx/sites-available/trendradar.aiyxtech.us.kg.conf`
- **启用方式:** 通过符号链接 `/etc/nginx/sites-enabled/trendradar.aiyxtech.us.kg.conf` 启用

#### 核心配置说明 / Core Configuration Details

**1. HTTP 到 HTTPS 重定向 (HTTP to HTTPS Redirect)**
```nginx
server {
    listen 80;
    server_name trendradar.aiyxtech.us.kg;
    return 301 https://$host$request_uri;
}
```
- **功能:** 将所有 HTTP 请求重定向到 HTTPS
- **状态码:** 301 (永久重定向)

**2. HTTPS 服务器配置 (HTTPS Server Configuration)**
```nginx
server {
    listen 443 ssl;
    server_name trendradar.aiyxtech.us.kg;
    
    # SSL 证书配置 / SSL Certificate Configuration
    ssl_certificate /etc/nginx/ssl/werss.crt;
    ssl_certificate_key /etc/nginx/ssl/werss.key;
}
```
- **监听端口:** 443 (HTTPS)
- **SSL 证书:** Cloudflare Universal SSL 通配符证书
- **证书路径:** `/etc/nginx/ssl/werss.crt` 和 `/etc/nginx/ssl/werss.key`

**3. 反向代理配置 (Reverse Proxy Configuration) - 关键部分**
```nginx
location / {
    proxy_pass http://127.0.0.1:8084;  # ⚠️ 必须与 docker-compose 端口映射一致（宿主机端口）
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    
    # 超时配置 / Timeout Configuration
    proxy_connect_timeout 300s;
    proxy_send_timeout 300s;
    proxy_read_timeout 300s;
    
    # WebSocket 支持 / WebSocket Support
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    
    # 缓冲配置 / Buffering Configuration
    chunked_transfer_encoding off;
    proxy_buffering off;
    proxy_cache off;
}
```

**说明 / Notes:**
- **端口映射关系**: Docker 容器通过 `/TrendRadar/docker/docker-compose.yml` 配置 `127.0.0.1:8084:8080`
  - 容器内部：Python `http.server` 自定义服务器监听 `8080` 端口
  - 宿主机映射：`127.0.0.1:8084` 映射到容器的 `8080` 端口
  - Nginx 代理：必须指向宿主机的 `8084` 端口（`proxy_pass http://127.0.0.1:8084`）
- **为什么不能直接访问容器端口**: 容器端口 `8080` 只在容器内部可访问，宿主机必须通过端口映射访问

#### 关键配置项说明 / Key Configuration Items

| 配置项 / Item | 值 / Value | 说明 / Description |
| :--- | :--- | :--- |
| `proxy_pass` | `http://127.0.0.1:8084` | **必须与 docker-compose 端口映射一致**（宿主机端口 8084 → 容器端口 8080） / Must match docker-compose port mapping (host 8084 → container 8080) |
| `proxy_connect_timeout` | `300s` | 连接超时时间，适合长时间数据处理 / Connection timeout for long-running operations |
| `proxy_send_timeout` | `300s` | 发送超时时间 / Send timeout |
| `proxy_read_timeout` | `300s` | 读取超时时间 / Read timeout |
| `proxy_buffering` | `off` | 禁用缓冲，实时转发响应 / Disable buffering for real-time response forwarding |
| `proxy_cache` | `off` | 禁用缓存，确保每次请求都到达后端 / Disable caching to ensure fresh data |

#### 常见问题与排查 / Troubleshooting

**问题 1: 502 Bad Gateway 错误**
- **原因:** `proxy_pass` 指向的端口与实际 docker-compose 端口映射不匹配
- **排查步骤:**
  1. 检查 docker-compose 端口映射: `docker ps | grep aiyxdata_tradar` 或查看 `/TrendRadar/docker/docker-compose.yml`
  2. 检查宿主机端口监听: `ss -tlnp | grep 8084`
  3. 检查 Nginx 配置中的 `proxy_pass` 端口
  4. 确保 Nginx 代理到宿主机映射端口（8084），而非容器内部端口（8080）
- **修复:** 更新 `proxy_pass` 为 `http://127.0.0.1:8084`，然后执行 `nginx -s reload`

**问题 2: 连接超时**
- **原因:** Python `http.server` 自定义配置服务器响应缓慢或未启动
- **排查步骤:**
  1. 检查容器是否运行: `docker ps | grep aiyxdata_tradar`
  2. 检查容器日志: `docker logs aiyxdata_tradar` 或 `docker logs -f aiyxdata_tradar`
  3. 检查网络连接: `curl http://127.0.0.1:8084/api/search?kw=test`
- **修复:** 启动容器 `docker-compose -f docker/docker-compose.yml up -d aiyxdata_tradar` 或增加超时时间

**问题 3: SSL 证书错误**
- **原因:** 证书文件不存在或路径错误
- **排查步骤:**
  1. 检查证书文件: `ls -la /etc/nginx/ssl/`
  2. 验证 Nginx 配置: `nginx -t`
- **修复:** 确保证书文件存在且路径正确

```javascript
function syncScroll(editorId, backdropId) {
```

```javascript
// 应用锁定状态 / Apply lock state
function applyLockState(isLocked) {
  const modifyButtons = ['btn-apply', 'btn-save-run'];
  const browseButtons = ['btn-reload', 'btn-load-default', 'btn-load-profile', 'btn-save-profile', 'btn-copy', 'theme-select'];
  const editors = ['yaml-editor', 'frequency-editor', 'timeline-editor', 'analysis_prompt-editor', 'translation_prompt-editor'];
  
  if (isLocked) {
    // 禁用修改按钮 / Disable modify buttons
    modifyButtons.forEach(btnId => {
      const btn = document.getElementById(btnId);
      btn.disabled = true;
      btn.style.opacity = '0.5';
      btn.style.cursor = 'not-allowed';
    });
    
    // 禁用输入元素 / Disable input elements
    document.querySelectorAll('input, select, textarea').forEach(elem => {
      elem.disabled = true;
      elem.style.pointerEvents = 'none';
      elem.style.opacity = '0.6';
    });
    
    // 设置编辑框为只读 / Set editors to read-only
    editors.forEach(editorId => {
      const editor = document.getElementById(editorId);
      editor.readOnly = true;
      editor.style.pointerEvents = 'auto';
      editor.style.backgroundColor = '#f5f5f5';
    });
  } else {
    // 恢复修改按钮 / Restore modify buttons
    modifyButtons.forEach(btnId => {
      const btn = document.getElementById(btnId);
      btn.disabled = false;
      btn.style.opacity = '1';
      btn.style.cursor = 'pointer';
    });
    
    // 恢复输入元素 / Restore input elements
    document.querySelectorAll('input, select, textarea').forEach(elem => {
      elem.disabled = false;
      elem.style.pointerEvents = 'auto';
      elem.style.opacity = '1';
    });
    
    // 恢复编辑框 / Restore editors
    editors.forEach(editorId => {
      const editor = document.getElementById(editorId);
      editor.readOnly = false;
      editor.style.backgroundColor = '#fff';
    });
  }
}

// 切换全局锁定 / Toggle global lock
function toggleGlobalLock() {
  isGlobalLocked = !isGlobalLocked;
  if (isGlobalLocked) applyLockState();
  
  // 更新按钮文本 / Update button text
  const lockBtn = document.getElementById('lock-toggle-btn');
  lockBtn.textContent = isGlobalLocked ? '锁定编辑' : '解除锁定';
}
```

- **加载方案:** `GET /api/profiles/load?name=<profileName>` → 返回完整方案数据
- **保存方案:** `POST /api/profiles/save`，body: `{ "name": "...", "content": { ...bundle } }` → 保存新方案或覆盖现有方案

### 3.3.7A 频率词编辑器实现约束 / Frequency Words Editor Constraints

#### 功能作用 / Purpose
- 将 `frequency_words.txt` 拆分为左侧源文本与右侧可视化卡片。
- 右侧卡片允许用户编辑全局过滤词、普通词组、组别名、单个别名、连续别名组与分类标签。
- 任意右侧变更最终都必须同步回左侧 `frequency-editor`，并由 APPLY 写入服务器 `config/frequency_words.txt`。

#### 解析与构建入口 / Parser & Builder
- **解析函数:** `parseFrequencyText(text)`，运行路径 `/TrendRadar/output/config_editor/assets/script.js`。
- **构建函数:** `buildFrequencyText(data)`，运行路径 `/TrendRadar/output/config_editor/assets/script.js`。
- **渲染函数:** `renderFrequencyPanel(data)`，运行路径 `/TrendRadar/output/config_editor/assets/script.js`。
- **同步函数:** `syncFrequencyToUI()` 与 `updateBackdrop('frequency-editor', 'frequency-backdrop')`。

#### 词组类型 / Group Types
- **`plain`:** 普通关键词组，一组多行关键词，空行分隔。
- **`group-name`:** 组别名，第一行为 `[组名]`，后续为关键词。
- **`alias`:** 单个别名，格式 `关键词或/正则/ => 显示别名`，前后由空行隔开。
- **`alias-group`:** 连续别名组，多行 `=>` 别名行之间无空行分隔。

#### 连续别名组强制 UI 约束 / Alias Group Mandatory UI Rules
- 连续别名组卡片必须显示标签 `连续别名组`。
- 列表标题必须说明 `别名列表（无空行分隔）`。
- 每个别名项包含：左侧关键词/正则、`=>`、右侧显示别名、删除按钮。
- **列表下方必须有 `+ 添加别名` 按钮**，按钮调用 `addAliasItem(groupIndex)`，并在同一张连续别名组卡片内新增一行。
- 新增空白别名项必须暂存在 `currentFrequencyData` 中并立即重新渲染 UI；空白项不能写入 `frequency_words.txt`，否则会生成非法的 ` => ` 行。
- 用户填写关键词后，`buildFrequencyText()` 才输出 `${keyword} => ${alias}`；若 alias 为空，可输出 `keyword => ` 以允许后续补别名。
- 删除别名项调用 `removeAliasItem(groupIndex, itemIndex)`；删除到 0 项时删除整组，删除到 1 项时降级为 `alias`。

#### 分类标签 / Tags
- 当前支持标签：`热榜`、`地区`、`财经`、`科技`、`文旅`、`教育`、`军事`、`国际`、`其他`。
- 标签写入格式为 `@tags:热榜,科技`，附加在对应词组末尾。
- `updateGroupTags(groupIndex, tag, isChecked)` 只改变当前组的 `tags` 数组，不应影响其它词组。
- **强制规则:** 每个关键词组必须至少选择一个标签。无标签时，右侧词组卡片顶部必须显示 `必须选择标签` 提示，并用缺失状态样式标出。
- **保存拦截:** `validateFrequencyGroupTags(content)` 必须在 `applyToBackend()`、`confirmSaveProfile()` 和 `saveFrequencyToLocalStorage()` 前执行；缺标签时要切换到 `frequency` 页并定位第一个缺标签词组。
- **后端兜底:** `/TrendRadar/docker/server.py` 的 `/api/save file=frequency` 必须调用 `validate_frequency_required_tags(content)`，直接 API 请求也不能绕过标签必选规则。
- **报告显示:** `/TrendRadar/aiyxdata_tradar/core/frequency.py` 必须解析 `@tags:`；`core/analyzer.py` 与 `report/generator.py` 必须把 `tags` 带入 `report_data["stats"]`；`report/html_v2.py` 必须在关键词卡片标题区显示所有标签，多个标签全部显示，不区分选择顺序。

### 3.3.7B 报告卡片来源标签约束 / Report Card Source Tag Constraints

#### 功能作用 / Purpose
- 报告卡片标题区必须能看出卡片来源类别：关键词聚合、热榜平台独立源、RSS 订阅独立源。
- 关键词聚合卡片使用 `frequency_words.txt` 中该词组的 `@tags:`；热榜平台与 RSS 卡片使用固定来源标签。

#### 固定标签规则 / Fixed Tag Rules
- 来自 `config.yaml` `platforms.sources`（第 2 节“数据源 - 热榜平台”）的独立卡片，必须固定显示 `热榜`。
- 来自 `config.yaml` `rss.feeds`（第 3 节“数据源 - RSS 订阅”）的独立卡片，必须固定显示 `RSS源`。
- 固定来源标签由 `/TrendRadar/aiyxdata_tradar/report/html_v2.py` 在构建 `all_blocks` 时按 `type` 写入：`source -> ["热榜", *tags]`，`rss -> ["RSS源", *tags]`。
- 该规则不得依赖源名称硬编码。无论未来新增、删除或重命名热榜平台/RSS 源，只要仍进入对应来源类型，就必须自动显示对应标签。

#### 手工标签规则 / Manual Tag Rules
- `config.yaml` 中每个 `platforms.sources[]` 和 `rss.feeds[]` 条目必须维护 `tags: [...]`，用于手工业务分类。
- 可选手工标签沿用频率词标签集合：`热榜`、`地区`、`财经`、`科技`、`文旅`、`教育`、`军事`、`国际`、`其他`；热榜平台因固定显示 `热榜`，手工选项中不再重复选择 `热榜`。
- 固定标签只说明来源类型，不满足手工标签必选要求；每个源必须至少有 1 个手工标签，可多选。
- 配置中心第 2/第 3 节源列表行必须显示固定标签、已选手工标签和 `标签` 多选下拉；标签变更后立即写回 YAML 并刷新行状态。
- `/api/save file=config` 与 `/api/profiles/save` 必须调用后端校验，不能只依赖前端 UI。
- 报告生成时，`/TrendRadar/aiyxdata_tradar/__main__.py` 必须从 `self.ctx.platforms` / `self.ctx.rss_feeds` 把对应源的 `tags` 写入 `standalone_data`，再由 `html_v2.py` 输出到卡片标题区。

#### 输出同步 / Output Sync
- 结构性实现必须优先修改 `/TrendRadar/aiyxdata_tradar/report/html_v2.py`。
- `/TrendRadar/output/index.html` 与 `/TrendRadar/output/html/latest/current.html` 是生成结果；若当前页面需要立即体现效果，可同步现有输出，但后续重新生成时仍以 `__main__.py` + `html_v2.py` 为准。

#### 易错点 / Common Pitfalls
- 报告弹窗里的配置中心不是 `docs/assets/script.js`，而是 `output/config_editor/assets/script.js`。
- 连续别名组新增按钮如果写在 `AI 写正则` 下方或分类标签下方，视觉上会被误解为整组操作；按钮应紧跟别名项列表之后、`AI 写正则` 行之前。
- `buildFrequencyText()` 不得无条件输出空 `item.keyword`，否则新增空行会污染配置文件并被解析器吞掉。
- 不要只在前端做标签必选；关键词组标签和数据源手工标签都必须由后端 `/api/save` / `/api/profiles/save` 同步验证，避免绕过可视化配置中心。
- 修改后必须运行 `node --check output/config_editor/assets/script.js`，并实际检查 `output/config_editor/index.html` 的脚本引用是否带新版本或可被刷新。

```javascript
async function testAIConnection() {
  const apiBaseInput = document.querySelector('input[data-path="api_base"]');
  const apiKeyInput = document.querySelector('input[data-path="api_key"]');
  const modelInput = document.querySelector('input[data-path="model"]');
  
  const apiBase = apiBaseInput ? apiBaseInput.value : '';
  const apiKey = apiKeyInput ? apiKeyInput.value : '';
  const model = modelInput ? modelInput.value : '';
  
  // 调用后端 API 测试连接
  const response = await fetch('/api/check_ai_connection', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      api_base: apiBase,
      api_key: apiKey,
      model: model || 'test'
    })
  });
  
  // 显示测试结果
  const result = await response.json();
  if (result.success) {
    showNotification('✅ 连接成功', 'success');

    showNotification('❌ 连接失败: ' + result.error, 'error');
  }
}
```

```javascript
async function fetchAvailableModels(apiBase, apiKey) {
  const modelListDiv = document.getElementById('modelList');

    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      api_base: apiBase,
      api_key: apiKey
    })
  });
  
  const result = await response.json();
  
  if (result.success && result.models && result.models.length > 0) {
    // 渲染模型列表
    const modelsHtml = result.models.map(model => `
      <button type="button" onclick="selectModel('${model}')" 
              class="p-3 border border-gray-300 rounded hover:bg-blue-50 hover:border-blue-500 text-left text-xs transition-colors">
        <div class="font-medium text-gray-800">${model}</div>
        <div class="text-gray-500 text-xs mt-1">点击选择</div>
      </button>

  } else {
    modelListDiv.innerHTML = `<div class="col-span-2 text-center text-red-600 py-4">
      <i class="fa-solid fa-exclamation-circle mr-2"></i>获取模型列表失败<br>
```

```javascript
function selectModel(modelName) {
  const modelInput = document.querySelector('input[data-path="model"]');
  if (modelInput) {
    const modelWithPrefix = autoAddProviderPrefix(modelName);
    modelInput.value = modelWithPrefix;
    modelInput.dispatchEvent(new Event('change', { bubbles: true }));
    
    // 应用模型配置规则，可能更新 api_base/temperature/max_tokens
    applyModelConfig(modelName);
    
    // 关闭模态框
    closeModelQueryModal();
    showToast(`已选择模型: ${modelWithPrefix}`, 'success');
  }
}
```

### 3.3.9 模型查询功能后端优化 / Model Query Backend Optimization

**功能概述 / Feature Overview**
优化后端 `/api/get_ai_models` 接口，支持多种 API 提供商的模型列表获取，增强容错性和兼容性。

**核心改进 / Core Improvements**

**1. 多格式模型提取支持 (Multi-format Model Extraction)**
- **文件位置:** `/TrendRadar/docker/server.py`，搜索 `/api/get_ai_models`
- **支持的容器字段:** `data`, `models`, `result`, `items`
- **支持的ID字段:** `id`, `model`, `name`, `model_id`
- **实现逻辑:**
  ```python
  # 支持多种容器字段名
  data_list = data.get('data') or data.get('models') or data.get('result') or data.get('items')
  
  if data_list and isinstance(data_list, list):
      for m in data_list:
          if isinstance(m, dict):
              # 支持多种ID字段名
              model_id = m.get('id') or m.get('model') or m.get('name') or m.get('model_id')
              if model_id:
                  models.append(model_id)
  ```

**2. 自动重试 /v1 路径 (Auto-retry /v1 Path)**
- **问题:** 某些 API 提供商的 `/models` 端点返回 HTML 而非 JSON
- **解决方案:** 当 `/models` 返回 404 或非 JSON 时，自动尝试 `/v1/models`
- **实现逻辑:**
  ```python
  if (response.status_code == 404 or not is_valid_json) and not base_url.endswith('/v1'):
      v1_url = f"{base_url}/v1/models"
      print(f"[DEBUG] 尝试 /v1 路径: {v1_url}", file=sys.stderr, flush=True)
      v1_response = requests.get(v1_url, headers=headers, timeout=10)
      v1_data = v1_response.json()
      response = v1_response
      data = v1_data
      is_valid_json = True
      print(f"[DEBUG] v1请求完成: status={v1_response.status_code}", file=sys.stderr, flush=True)
  ```

**3. 改进的错误处理 (Improved Error Handling)**
- **避免双重 JSON 解析:** 检查 `is_valid_json` 标志，避免对非 JSON 响应调用 `.json()`
- **详细错误信息提取:** 从响应中提取 `message` 或 `code` 字段
- **实现逻辑:**
  ```python
  if is_valid_json:
      # 使用已解析的 data 变量
      err_msg = data.get('message') or data.get('code') or str(data)
  else:
      # 使用 response.text 而不是再次调用 json()
      err_msg = response.text[:200]
  
  return self.send_json_response(200, {
      "success": False,
      "error": f"API 返回错误 ({response.status_code}): {err_msg}"
  })
  ```

### 3.3.10 界面细节优化与错误修复 / UI Detail Optimization & Bug Fixes

**1. 字符转义优化 (Character Escaping Optimization)**
- **问题 (Problem):** 当关键词包含正则字符（如 `\b`）或引号时，直接插入 HTML 属性会导致转义丢失，表现为模态框中出现退格符乱码或 HTML 结构崩溃。/ When keywords contain regex characters (e.g., `\b`) or quotes, inserting them directly into HTML attributes caused loss of escaping, resulting in garbled characters or broken HTML.
- **实现细节 (Implementation):**
  - 新增 `escAttr()` 辅助函数，对反斜杠 `\`、单引号 `'`、双引号 `"` 和换行符进行多重转义。/ Added `escAttr()` helper to escape backslashes, quotes, and newlines.
  - 在 `renderGroupCard` 和 `renderFrequencyPanel` 中统一调用。/ Unified calling in rendering functions.
- **代码参考 (Code Reference):** `/TrendRadar/output/config_editor/assets/script.js`，搜索 `escAttr`、`renderFrequencyPanel`。

**2. 标签栏自适应优化 (Tab Bar Responsive Optimization)**
- **问题 (Problem):** 五个文件标签过多且 AI 提示词文件名较长时，在左侧编辑区内容易发生挤压、遮挡或第五个标签不可见。/ The five file tabs include long AI prompt filenames, so the left editor area can squeeze, hide, or clip the fifth tab.
- **实现细节 (Implementation):**
  - DOM：标签栏容器使用 `.file-tabbar`，五个文件按钮都使用 `.tab-button`；按钮内部包含 `.tab-file` 英文文件名与 `.tab-label` 中文说明两行，并保留 `title` 作为完整文件名提示。/ DOM: The tab bar uses `.file-tabbar`; each `.tab-button` contains `.tab-file` and `.tab-label` lines, with `title` for full filenames.
  - CSS：五个分页签等宽铺开，采用页签式圆角顶边、激活态底部强调线；`.file-savebar` 独立承载保存状态。/ CSS: Five paged tabs are evenly distributed with rounded top corners and active underline; `.file-savebar` carries save status separately.
  - JS：`fitFileTabs()` 在初始化、窗口 resize 和 `switchTab()` 后重新计算每个分页签可用宽度，按最长英文文件名缩放 `.tab-file` 和 `.tab-label` 字号。/ JS: `fitFileTabs()` recalculates per-tab width after init, resize, and tab switching, scaling `.tab-file` and `.tab-label` based on the longest English filename.
  - 缓存：修改脚本后更新 `output/config_editor/index.html` 中 `./assets/script.js?v=...`，避免 iframe 继续加载旧逻辑。/ Cache: Update the `./assets/script.js?v=...` reference after script changes so the iframe does not keep old logic.
- **强制约束 (Mandatory Constraint):** 生产配置中心必须能看到五个分页签，不得只显示前四个或依赖用户横向滚动找第五个；英文文件名不得被省略号截断；不要为解决标签栏问题改动读取、保存、APPLY、SAVE & RUN 或右侧面板逻辑。
- **代码参考 (Code Reference):** `/TrendRadar/output/config_editor/index.html` 搜索 `.file-tabbar`；`/TrendRadar/output/config_editor/assets/style.css` 搜索 `.tab-button`；`/TrendRadar/output/config_editor/assets/script.js` 搜索 `fitFileTabs`。

**3. 编辑器背景主题一致性 (Editor Background Theme Consistency)**
- **问题 (Problem):** 编辑器外层容器硬编码了 `#1e1e1e` 背景，导致在切换“北欧蓝 (Nord)”或“德古拉 (Dracula)”等彩色背景主题时，空文本区域颜色与高亮层背景色（`var(--bg-editor)`）不一致。/ Hardcoded `#1e1e1e` background on containers caused inconsistency with the highlight layer's theme-based color (`var(--bg-editor)`) in themes like Nord or Dracula.
- **实现细节 (Implementation):**
  - 移除 `index.html` 中所有编辑器的硬编码 `bg-[#1e1e1e]` 类。/ Removed all hardcoded `bg-[#1e1e1e]` classes from `index.html`.
  - 在 `style.css` 的 `.highlight-editor-wrap` 类中统一使用 `background: var(--bg-editor);`。/ Unified using `background: var(--bg-editor);` in `.highlight-editor-wrap` in `style.css`.
- **效果 (Effect):** 编辑器背景色现在能够随主题切换实时、全域更新。/ Editor background now updates globally and in real-time with theme switching.
